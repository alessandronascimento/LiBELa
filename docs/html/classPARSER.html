Detailed Description

The PARSER class is used to get the input parameters from the input file. These parameters are used in almost every class of the program.

Member Data Documentation

bool PARSER::flex_lig

Flag to define if the ligand will be defined as flexible. The flag is a automatically turned on when the input command lig_traj is used.

bool PARSER::flex_rec

Flag to define if the receptor will be defined as flexible. The flag is a automatically turned on when the input command rec_traj is used.

string PARSER::lig_traj

Filename with ligand "trajectory" to use it in a semi-flexible approach. The file format is the same as in AMBER trajectory.

string PARSER::ligand_energy_model

There are two currently implemented functions for evaluating the ligand internal energy. The possible choices are: \"gaff\" or \"MMFF94\"

string PARSER::overlay_optimizer

Algorithm for overlay optimization. Current options are: lbfgs ln_auglag cobyla mma subplex ld_auglag crs direct

string PARSER::rec_traj

Filename with receptor "trajectory" to use it in a semi-flexible approach. The file format is the same as in AMBER trajectory.

int PARSER::scoring_function

Defines the scoring function to be used throughout the code. 0 = Softcore + Solvation 1 = Softcore 2 = Amber FF + Solvation 3 = Amber FF

bool PARSER::sort_by_energy

Sort conformers using initial binding energy. If set to false, conformers will be ranked by Overlay objective function.

int PARSER::timeout

Defines a timeout function (in seconds) in Equilibrium simulations. Above this limit, the program will abort the execution.
